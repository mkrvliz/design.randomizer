# Topic Randomizer — с интеграцией Yandex GPT

Генератор креативных идей проектов на основе нейросети Яндекса.

## Установка

### 1. Установи Node.js
Скачай и установи [Node.js](https://nodejs.org/) (версия 16+)

### 2. Установи зависимости
Открой терминал в папке проекта и выполни:
```bash
npm install
```

### 3. Настрой Yandex Cloud API

**Шаг 1: Создай аккаунт Yandex Cloud**
- Перейди на [console.cloud.yandex.com](https://console.cloud.yandex.com/)
- Зарегистрируйся или войди в аккаунт

**Шаг 2: Создай новый проект (Folder)**
- В консоли создай новую папку (folder)
- Скопируй её ID — это понадобится

**Шаг 3: Создай API ключ**
- В консоли Yandex Cloud перейди в "Service Accounts"
- Создай новый сервисный аккаунт
- Создай для него API ключ
- Скопируй ключ — это понадобится

**Шаг 4: Дай права доступа**
- В настройках папки (folder) добавь этому сервисному аккаунту роль "ai.languageModels.user"

### 4. Создай .env файл
В корне проекта переименуй файл `.env.example` в `.env` и заполни его:

```
YANDEX_API_KEY=your_api_key_here
YANDEX_FOLDER_ID=your_folder_id_here
PORT=3000
```

Вставь свои значения вместо `your_api_key_here` и `your_folder_id_here`

## Запуск

### Запусти сервер:
```bash
npm start
```

Сервер запустится на `http://localhost:3000`

### Открой приложение:
- Открой браузер и перейди на `http://localhost:3000`
- Нажимай на кнопку "Выдать тему" — новые идеи будут генерироваться через нейросеть!

## Как это работает

1. Ты нажимаешь кнопку на сайте
2. Фронтенд отправляет запрос на бэкенд-сервер
3. Бэкенд отправляет запрос в API Яндекса
4. Yandex GPT генерирует новую идею проекта
5. Результат возвращается на фронтенд и отображается

## Troubleshooting

**"Cannot find module 'express'"**
- Выполни: `npm install`

**"Server running but page won't connect"**
- Убедись, что сервер запущен (`npm start`)
- Проверь URL в браузере: `http://localhost:3000`

**"Ошибка подключения к серверу" на странице**
- Убедись, что сервер запущен
- Проверь, что `.env` файл правильно заполнен
- Проверь консоль браузера (F12) на ошибки

**"401" ошибка от Yandex API**
- Проверь правильность API ключа в `.env`
- Проверь, что у сервисного аккаунта есть роль "ai.languageModels.user"

**"403" ошибка**
- Проверь что FOLDER_ID правильный
- Проверь что у сервисного аккаунта есть доступ к этой папке
